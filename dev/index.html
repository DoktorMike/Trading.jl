Trading.Cash — Type
Cash

Represents the actual cash balance. Currently there is no particular currency tied to this.

source
Trading.PurchasePower — Type
PurchasePower

Represents the current purchasing power. This is updated at the start of each update cycle to the current value of the Cash singleton. It can thus be used to determine how many purchases/trades can be made during one cycle.

source
Trading.Position — Type
Position

Represents a position held in an equity represented by ticker.

source
Trading.Purchase — Type
Purchase

The local representation of a purchase order. This will be turned into an Order by the Purchaser System as soon as it's communicated to the broker.

source
Trading.Sale — Type

The local representation of a sell order. This will be turned into an Order by the Seller System as soon as it's communicated to the broker.

source
Trading.Order — Type
Order

Representation of a Purchase or Sale order that has been communicated to the broker. Once the status goes to "filled" the filling information will be taken by the Filler System to create a Filled component.

source
